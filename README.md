# lecture-pagewise-explainer-skill

一个用于逐页讲解课件、讲义和教学幻灯片的 Codex skill。

## 模型推荐

本 skill 推荐用于 **GPT-5.5** 模型，以获得更稳定的逐页理解、结构化解释和长文档覆盖效果。

对于其他模型，目前尚未做充分试验；如果你希望获得接近理想的效果，可能需要根据自己的课件类型、学科背景和输出偏好做一些微调。

## 适用场景

当你需要让 Codex 按页解释教学 PDF、课件截图或课程幻灯片时，可以使用这个 skill。它强调：

- 逐页对应，不跳页，不漏页
- 保持页面顺序
- 英文内容翻译成自然中文，中文原文尽量保留
- 每一页都覆盖知识点、公式、符号、图示、例题和习题提示
- 对重复页、动画递进页，只解释新增内容
- 在合适的地方引用原文：需要逐字保留时引用原文，讲解依赖图示、图表、表格或复杂公式时插入对应页面/局部截图

这个仓库保留了 skill 本体和其默认配置：

- `SKILL.md`
- `agents/openai.yaml`

## 如何使用

在对话中引用这个 skill，并提供课件 PDF 或截图。适合的提问方式例如：

- `请用 lecture-pagewise-explainer-skill 逐页讲解这份课件`
- `按页解释这组幻灯片，不要漏公式`
- `这份 PDF 每一页都翻译并解释`

如果你只是想要整套课件的总体脉络，而不是逐页对应解释，那么更适合改用 synthesis 类 skill。

## 给其他专业同学的修改指南

这个 skill 的核心能力是“逐页讲解 + 严格页码对应”，它本身并不绑定计算机、信息论或数学课程。不同专业的同学在使用时，通常只需要改写它的说明和示例，而不是改掉它的工作流。

### 1. 保留不变的部分

以下部分通常可以原样保留：

- “逐页讲解”“不漏页”“动画递进只解释新增内容”这类流程要求
- 对公式、符号、图示、例题、习题的覆盖要求
- 对重复页的压缩解释规则
- 对中英文混排的翻译规则
- 在合适的地方引用原文（文字引用或截图）的规则

这些要求与学科无关，适用于大多数课程材料。

### 2. 需要按专业替换的部分

如果你的课件不是计算机或信息论相关，可以把 skill 里的示例表达和措辞替换成你自己的专业语境。

例如：

- 计算机专业：把“算法、数据结构、复杂度、状态转移”作为常见例子
- 数学专业：把“定义、定理、推导、证明、反例”作为常见例子
- 物理专业：把“公式、实验装置、变量含义、物理量单位”作为常见例子
- 经济/管理专业：把“模型假设、图表、指标、案例分析”作为常见例子
- 文科专业：把“概念定义、论证结构、文本对照、术语解释”作为常见例子

### 3. 建议你改写的地方

如果你要把它改成自己的专业版，优先改这几处：

- `SKILL.md` 的描述和示例关键词
- `agents/openai.yaml` 里的 `display_name` 和 `default_prompt`
- README 里的示例提问语句

例如，经济学方向可以改成：

- `请用这个 skill 逐页讲解经济学课件`
- `把这份讲义每一页都翻译并解释，尤其不要漏掉图表和公式`

### 4. 不建议改动的地方

不建议随意删掉这些规则：

- 逐页对应
- 页面重复内容去重
- 只在必要时合并说明
- 不确定内容要明确说明“看不清”或“无法识别”

这些是这个 skill 的核心质量约束。

### 5. 如果你要做成自己的衍生版

可以把这个仓库 fork 后，把专业名写进标题，例如：

- `math-pagewise-explainer-skill`
- `economics-pagewise-explainer-skill`
- `physics-pagewise-explainer-skill`

然后只替换示例和提示词，不必重写整个技能框架。
